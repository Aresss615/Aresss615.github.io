C-01 Production · Retail Point-of-Sale

J&J Grocery POS

A complete point-of-sale and inventory system running the day-to-day of a real Philippine grocery store.

Problem

The store ran sales and stock by hand — slow checkout, no audit trail, and no reliable view of what was selling or what was left on the shelf.

Approach

Vanilla PHP + MySQL, no framework — so it deploys on any cheap PHP host the store can actually afford. Barcode checkout auto-decrements stock; every sale, void, and refund posts to a double-entry accounting journal and an immutable audit trail. Receipts follow BIR formatting with 12% VAT and cash, GCash, and card tender.

Engineering depth

Refunds use optimistic locking so double-refunds are impossible. Access is permission-based across four roles. CSRF protection on every write, session timeout, and Z-Read day-close. The schema has evolved across nine migrations as the store's needs grew, with PHPUnit tests on the core flows.

C-02 Product · Configurable SaaS

BookmeIN — Reservation Platform

One booking codebase that reskins itself for hotels, restaurants, clinics, or salons.

Problem

Small service businesses all need the same engine — take bookings, never double-book — but with different vocabulary and fields.

Approach

A business-type preset drives dynamic labels and booking fields, so one install becomes a hotel-room booker or a dental scheduler by flipping a setting. Vanilla PHP API behind a single-file JS SPA — no build step.

Engineering depth

Overbooking is prevented at the database level — bookings commit inside a transaction using SELECT … FOR UPDATE, so two customers can't grab the same slot. bcrypt + HttpOnly session cookies, role-based access, and KPI reporting.

C-03 Open Source · AI Media Pipeline (Python)

MoneyPrinterTurbo — Extended

An open-source TikTok / Shorts generator I forked and extended — turning a script into a finished, captioned vertical video, end to end.

Problem

The base project picked stock clips at random and leaned on a paid cloud voice. Captions were block-timed, so words never lit up in sync with the narration — the videos felt generic and the costs didn't scale.

Approach

I forked MoneyPrinterTurbo and rebuilt three subsystems. Semantic clip matching embeds the script and ranks stock footage by meaning instead of random selection. Local Chatterbox TTS with voice cloning replaces paid Azure — no API cost, no rate limits. And captions highlight word-by-word, lighting each word exactly as it's spoken.

Engineering depth

Word-level highlighting is driven by WhisperX forced-alignment against the TTS audio, so subtitles track real word boundaries across wrapped, multi-line text. The render path runs on MoviePy with automatic CPU fallback when CUDA is unavailable — built to run on whatever machine is in front of you.
